# 架构设计

Memabra 采用分层架构，将直觉判断、记忆管理、执行反馈三个核心环节解耦。

## 核心组件

### 1. Intuition Network (直觉网络)

**职责**：快速模式识别，决定"第一反应"

**输入**：
- 问题的嵌入向量 (embedding)
- 可选：上下文摘要

**输出**：
- `strategy_id`: 选择哪种解决策略
- `confidence`: 置信度 (0-1)
- `memory_query`: 构造的记忆检索查询

**实现选项**：
- **轻量版**：余弦相似度 + 策略向量库
- **进阶版**：小型 MLP (2-3层)
- **高级版**：基于 Transformer 的轻量编码器

### 2. Memory System (记忆系统)

采用三层结构，对应人脑记忆类型：

| 层级 | 对应生物记忆 | 内容 | 存储形式 |
|------|--------------|------|----------|
| Episodic | 情景记忆 | 具体事件、对话历史 | 时间序列 + 向量索引 |
| Semantic | 语义记忆 | 事实、概念、关系 | 知识图谱 + 向量库 |
| Procedural | 程序记忆 | 技能、方法、工作流 | 规则 + 参数化表示 |

### 3. Executor (执行器)

根据直觉网络选择的策略和检索到的记忆，执行具体动作：
- 工具调用（搜索、计算、API）
- 代码执行
- 直接生成回复

### 4. Implicit Evaluator (隐式评估器)

**核心设计**：不打扰用户，从自然交互中读取满意度信号

评估维度：
- **即时信号**：用户下一条消息的语义、情感、意图
- **短期信号**：对话是否继续、是否追问同一问题
- **长期信号**：会话是否成功结束、该记忆是否被再次调用

### 5. Learning Loop (学习循环)

将评估结果反馈给直觉网络，实现自我优化：
- 正反馈 → 提高该 (problem, strategy) 配对的概率
- 负反馈 → 降低概率，同时触发探索机制

## 数据流

```
1. 接收问题 → 2. 直觉判断 → 3. 召回记忆 → 4. 执行 → 5. 等待反馈
                                              ↓
6. 观察后续交互 → 7. 计算奖励 → 8. 更新网络 → 9. (可选) 深度搜索
```

## 关键技术选型

| 组件 | 推荐方案 | 备选方案 |
|------|----------|----------|
| 向量存储 | Chroma / Milvus | Pinecone / Weaviate |
| 嵌入模型 | sentence-transformers | OpenAI embedding |
| 轻量网络 | PyTorch (small MLP) | NumPy + 手动实现 |
| 图存储 | NetworkX (轻量) | Neo4j (大规模) |
| 日志/追踪 | 本地 JSON / SQLite | LangSmith / W&B |

## 扩展性考虑

- **水平扩展**：记忆层可替换为分布式向量数据库
- **多 Agent**：支持 Agent 间记忆共享（社会记忆）
- **多模态**：未来支持图像、音频记忆
